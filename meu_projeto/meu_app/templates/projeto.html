Características do Projeto

Aplicação Desktop

-> O sistema principal do projeto foi realizado na aplicação desktop, desenvolvido em Python com aplicação visual em PyQt5. O aplicativo inclui um sistema completo de cadastro de veiculos e usuários, vizualização de vagas disponíveis, além de outros tipos de serviços e dentre outras aplicações.

COMO FOI ESTRUTURADA?

  • Modelo MVC (Model - View - Controller).
  • Possui um sistema de autenticação completo.
  • Querys de SQL do Banco de Dados para aplicação e armazenamento dos dados.
  • Link para o repositório: github.com/DashVision/gurgel-park-desktop

ALGUMAS BIBLIOTECAS USADAS:

  • BRCRYPT - Usada para criptografar senhas de forma segura
  • DOTENV - Usada para carregar variáveis de ambiente de um arquivo '.env' permitindo guardar informações sensíveis ou de configuração fora do código
  • PYDANTIC - Criar modelos de dados estruturados

Aplicação Web

-> A aplicação web demonstrativa foi desenvolvida através do framework Django com o proposito de integrar o backend e o frontend sem complicações, além disso o site foi estrutura e estilizado em HTML5 e CSS3

-> O propósito do Django foi desenvolver um processo de autenticacao de Cadastro e Login, permitindo o acesso ao site apenas para quem fosse registrado.

Estruturação

  • Sistema de rotas (URLS) - '127.0.0:1800/auth/projeto/.
  • Métodos Users e Autenticate.
  • Respostas em HttpResponse.
  • Métodos GET e POST.

Propósito

  • Desmonstração do projeto
  • Exemplos de aplicação do DJANGO
  • Demonstração da utilização HTML com CSS DJANGO
  • Link para o repositório: github.com/DashVision/Web-Gurgel-Park

Integração com Microcontrolador

-> A parte de Microcontrolador foi a responsável pela coleta de dados do projeto para o funcionamento do sistema. Nessa parte foi utilizado um ESP32 e um sensor HC-SR04.

-> Há também uma certa integração com o banco de dados da linguagem SQL, entretanto essa parte está de certa forma implementada no sistema desktop.

ESTRUTURA

  • Sensor HC-SR04 - Sensor de distância, possível detectar com certa precisão até 4 metros.
  • ESP32 - Microcontrolador com baixo consumo de energia e com conectividade com Wi-Fi , ideal para o nosso projeto
  • Arduino IDE - IDE utilizada para montar o algortimo responsável pela interpretação dos dados.

CONSTRUÇÃO

  • O sensor fica posicionado de tal forma para detectar o veículo.
  • O ESP32 e o sensor funcionam de forma autonoma e a informação é enviada através do WiFi
  • O algortimo reconhece o veículo através da logica de distância do sensor.
  • Os dados são enviados para o banco de dados para o tratamento.
